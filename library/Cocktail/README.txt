Задача: имеется базовый самостоятельный процесс (например, создание некоторого стандартного коктейля), но должна
быть опциональная возможность расширения этого процесса независимыми компонентами (создань коктейль на молоке,
создать коктейль с банановым вкусом, создать коктейль с банановым вкусом на молоке и т.д.). Также данный процесс
должен быть вызван за один присест, а не в несколько действий ( 1) создать коктейль, 2) добавить молоко,
3) добавить банан )

1) В процедурном стиле для решения задачи можно было бы создать функцию с большим количеством опциональных
необязательных параметров:

function getCoctail($substance = 'water', $taste = false) {...}

Неудобство заключается в нарушении первого приципа SOLID, а также в невозможности гибко манипуляровать параметрами
(для создания коктейля с банановым вкусом придется прописывать явно параметр по умолчанию $substance)

2) В ООП стиле задача решается с помошью паттерна проектирования "декоратор". Данный пакет классо как раз его и
реалицует. Примеры использования:

- простой коктейль
    $a = new Cocktail\Protein();
    $a->getCocktail();

- коктейль на молоке
    $b = new Cocktail\Milk (new Cocktail\Protein());
    $b->getCocktail();

- коктейль с банановым вкусом
    $c = new Cocktail\Banana(new Cocktail\Protein());
    $c->getCocktail();

- коктейль на молоке с банановым вкусом
    $d = new Cocktail\Milk (new Cocktail\Banana(new Cocktail\Protein()));
    $d->getCocktail();
